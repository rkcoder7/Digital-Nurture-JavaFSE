Exercise 19.2 – Custom Sass Variables

This page demonstrates the visual result of overriding Bootstrap's Sass variables. The hot-pink primary, purple secondary, teal success, and extra-rounded corners are all controlled by _variables.scss overrides.

📄 scss/_variables-override.scss 
// ── COLOR OVERRIDES ──────────────────────────────────────────
// These override Bootstrap's $primary, $secondary, etc.
// Must be defined BEFORE @import "bootstrap"

$primary:         #e83e8c;   // hot pink  (default was #0d6efd blue)
$secondary:       #6f42c1;   // purple    (default was #6c757d grey)
$success:         #20c997;   // teal      (default was #198754 green)
$danger:          #fd7e14;   // orange    (default was #dc3545 red)
$warning:         #ffc107;   // unchanged
$info:            #0dcaf0;   // unchanged

// ── SPACING & SIZING ─────────────────────────────────────────
$border-radius:       1rem;   // more rounded (default: 0.375rem)
$border-radius-sm:    0.6rem;
$border-radius-lg:    1.5rem;
$border-radius-pill:  50rem;

// ── TYPOGRAPHY ───────────────────────────────────────────────
$font-family-base:    'Georgia', serif;   // default: system-ui
$font-size-base:      1.05rem;            // default: 1rem
$headings-font-weight: 700;               // default: 500
$line-height-base:    1.7;               // default: 1.5

// ── BODY ─────────────────────────────────────────────────────
$body-bg:    #f8f0fb;    // light lavender background
$body-color: #2d2d2d;    // slightly softer than pure black

How it works in Sass
  1. You define $primary: #e83e8c in your _variables-override.scss.
  2. Bootstrap checks: "is $primary already defined?" — Yes → uses yours.
  3. All Bootstrap components that reference $primary now use your value.
  4. Recompile once → every button, badge, alert, link inherits the new color.
